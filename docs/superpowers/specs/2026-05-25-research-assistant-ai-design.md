# Research Assistant AI Agent 系统设计文档

> 日期：2026-05-25
> 目标：构建以 Claude Code 为基座的汽车与电池材料研究助理 AI 系统

---

## 一、设计目标

**长期愿景**：可扩展的多智能体研究助理框架，完整覆盖数据采集、新闻监控、财务建模、报告生成、投关协同全流程。

**Phase 1 目标（起步阶段）**：
- 验证核心场景最小闭环：新闻监控 → 数据更新 → 快评生成 → 分析师审批 → 归档
- 建立 Skills/MCP/Agent 基础能力层
- 搭建文件+Obsidian+向量库三位一体存储架构

## 三、核心业务流程（Phase 1）

### 3.1 触发方式

用户在 Claude Code 输入自然语言指令启动任务：
```
/research 今天电池材料 news
/data update CATL 2024Q1
/report 快评 宁德时代 江西锂矿
/review [粘贴新闻文本] → AI 生成快评
```

### 3.2 标准流程

```
[触发] 用户指令
   ↓
[意图识别] Skill 判断任务类型（新闻/数据/报告）
   ↓
[News经纪] MCP获取今日新闻 → NER提取公司/产品/政策
   ↓
[Data经纪] 自动更新本地数据文件 → 规则校验 → 异常告警
   ↓
[Report经纪] 生成结构化快评草稿
   ↓
[审批] Claude Code 展示草稿 → 用户确认/修改/驳回
   ↓
[归档]
   ├→ 本地文件（原始数据/草稿/最终版/版本快照）
   ├→ Obsidian（新笔记/双向链接/模板填充）
   └→ 向量库（语义索引/上下文检索）
```

### 3.3 审批节点设计

| 节点 | 操作 | AI行为 |
|------|------|--------|
| 草稿生成完成 | 展示草稿，等待确认 | 输出结构化内容 + 关键问题列表 |
| 用户修改 | 提供修改意见 | 即时更新草稿，重新展示 |
| 用户驳回 | 说明原因 | 记录反馈，重新执行或人工处理 |
| 用户通过 | 确认发布 | 执行归档，更新索引 |

## 四、存储架构

### 4.1 目录结构

```
p39_VoltInsight/
├── data/                      # 原始数据文件
│   ├── raw/                   # 原始数据（财报/价格/装机量）
│   ├── processed/             # 清洗后数据
│   └── archive/                # 历史归档
├── research/                   # 研究产出
│   ├── drafts/                 # 草稿
│   ├── final/                  # 定稿
│   └── versions/               # 版本快照
├── knowledge/                  # 知识库
│   ├── templates/              # Obsidian 模板
│   ├── notes/                  # 笔记
│   └── graph/                  # Obsidian 双向链接
└── vector_db/                  # 向量库（ChromaDB）
```

### 4.2 Obsidian 集成策略

- **模板驱动**：快评/纪要/报告使用统一 Obsidian 模板
- **双向链接**：公司→新闻→快评→数据，形成知识图谱
- **元数据**：每个笔记包含日期、类型、状态、标签等字段
- **每日笔记**：建立日期索引笔记，汇总当日产出

### 4.3 向量库策略

- **索引内容**：快评正文、公司简介、行业框架、专家观点
- **检索用途**：上下文增强（RAG）、相似案例推荐
- **更新机制**：归档时同步写入向量库

## 五、Skills 设计

### 5.1 核心 Skill 列表

| Skill | 用途 | 触发指令示例 |
|-------|------|-------------|
| `research-news` | 新闻监控与快评 | `/research 今天 news` |
| `research-data` | 数据更新与校验 | `/data update CATL Q1` |
| `research-report` | 报告章节生成 | `/report 快评 公司` |
| `research-review` | 快速内容审查 | `/review <文本>` |

### 5.2 Skill 结构

每个 Skill 包含：
- **System Prompt**：角色定义 + 领域知识 + 输出规范
- **Tools/MCP**：该领域需要调用的数据源
- **Output Format**：结构化输出模板
- **Quality Gate**：质量检查清单

## 八、关键设计决策

### 8.1 HITL 强制节点

**全人工**：投资观点定调、评级/目标价确定、敏感信息判断、最终发布审批

**AI辅助**：数据清洗、格式排版、新闻去重、基础图表生成、草稿撰写

### 8.2 计算隔离原则

所有数学计算（财务建模、估值、指标计算）必须通过 Python 代码执行，LLM 仅负责逻辑描述与结果解释。

### 8.3 知识增强策略

建立历史研报+技术路线图谱+Rationale 库，作为 RAG 检索增强，防止 AI 输出浅层化。

## 十一、风险与应对

| 风险 | 应对 |
|------|------|
| 新闻源不稳定 | MCP外置，支持切换数据源 |
| AI输出浅层化 | RAG知识增强，分析师反馈迭代 |
| 数据幻觉/计算错误 | 强制Python代码执行计算 |
| 合规风险 | 内置披露模板，发布前人工审批 |
