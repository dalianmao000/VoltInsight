## 系统架构

```
┌─────────────────────────────────────────────────────────┐
│                      用户交互层                             │
│              Claude Code + Obsidian + 本地文件              │
└───────────────────────┬─────────────────────────────────┘
                        │
┌───────────────────────▼─────────────────────────────────┐
│                    编排层 (Orchestrator)                    │
│              LangGraph 工作流 + APScheduler 定时器         │
└───────────────────────┬─────────────────────────────────┘
                        │
┌───────────────────────▼─────────────────────────────────┐
│                      能力层 (Skills)                       │
│             新闻 │ 数据 │ 报告 │ 审查                      │
└───────────────────────┬─────────────────────────────────┘
                        │
┌───────────────────────▼─────────────────────────────────┐
│                      存储层 (Storage)                       │

└─────────────────────────────────────────────────────────┘
```

## 技术栈

- **Python 3.11+**
- **LangGraph** - 工作流编排
- **APScheduler** - 定时调度
- **SQLAlchemy** - 数据库 ORM
- **ChromaDB** - 向量数据库
- **Anthropic API** - 大语言模型

## 安装

```bash
pip install -r requirements.txt
```

## 快速开始

```bash
# 初始化数据库
python -m src.main --init-db

# 手动执行工作流
python -m src.main --workflow daily_briefing

# 启动定时调度器
python -m src.main --start-scheduler
```

## 项目结构

```
src/
├── orchestrator/      # 工作流管理 (LangGraph, Scheduler)
├── db/               # 数据库模型和仓储
├── services/         # 通知服务
├── skills/           # 研究技能 (新闻/数据/报告/审查)
├── storage/         # 存储层 (文件/Obsidian/向量库)
├── mcp/             # MCP 数据源连接器
└── main.py          # 入口文件

tests/                # 单元测试和集成测试
knowledge/           # Obsidian 笔记模板
docs/                # 设计文档和规格说明
```

## 工作流

- **每日早报**: 新闻 + 数据更新 → 报告生成 → 审批 → 归档
- **新闻快评**: 新闻获取 → 快评撰写 → 审批 → 归档
- **数据更新**: 批量数据更新 → 校验 → 归档
